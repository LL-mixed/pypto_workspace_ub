# `simpler` 接入计划配套模板：阶段结果记录

## 1. 文档目的

本模板用于在每个里程碑结束后，按统一口径记录阶段结果。

目标：

1. 固定阶段结果记录格式
2. 固定“通过 / 不通过 / 带风险通过”的表达方式
3. 固定问题、风险、阻塞项的记录位置

本模板用于记录结果，不用于替代设计文档和验证矩阵。

---

## 2. 使用方式

每个里程碑结束时，复制本模板生成一份阶段结果记录。

建议命名方式：

`simpler_integration_<milestone>_result_<YYYYMMDD>.md`

示例：

`simpler_integration_m1_result_20260414.md`

## 4. 结论填写规则

### `PASS`

仅在以下条件同时满足时可使用：

1. 阶段目标基本完成
2. 核心验证已通过
3. 没有阻止进入下一里程碑的关键缺口

### `FAIL`

以下情况应使用：

1. 阶段核心目标未完成
2. 核心验证未通过
3. 存在关键阻塞且未解除

### `PASS WITH RISKS`

以下情况可使用：

1. 阶段目标基本完成
2. 可以进入下一阶段
3. 但存在必须显式暴露的已知风险

固定规则：

1. `PASS WITH RISKS` 不能拿来掩盖核心失败
2. 若核心验证未通过，不得写 `PASS WITH RISKS`

---

## 5. 不接受的记录方式

以下方式都不接受。

1. 只有“基本完成”“差不多可用”这类模糊描述
2. 不写未完成项
3. 不写验证摘要直接下结论
4. 不区分风险和阻塞
5. 不区分“能继续推进”和“已经完成”

---

## 6. 使用建议

1. 每个里程碑都生成独立结果记录
2. 阶段结果记录应由该阶段收口 owner 主写
3. Validation Owner 应参与“验证结果摘要”和“是否允许进入下一阶段”的结论确认
4. 架构 Owner 应参与最终放行判断
